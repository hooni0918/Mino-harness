# 병렬 적대 에이전트로 발전시키기

이 번들은 한 번 만들고 끝이 아니라, **여러 에이전트가 서로를 공격하게 해서** 결함을 뽑아내며 단단해진다.
혼자 검토하면 자기가 쓴 논리의 빈틈을 못 본다. 차원을 나눠 동시에 공격하고, 살아남은 것만 채택한다.

grep으로 끝나는 결정론적 정합(호명 오류·죽은 링크·axe 명령 존재·금지 문자열)은 `scripts/check_consistency.py`가
CI에서 기계로 잡는다. 여기 적대 에이전트는 판단이 필요한 것 — "문서가 약속한 게이트가 코드에도 있는가" 같은
의미론적 대조만 맡는다.

## 왜 "적대"인가

한 명의 리뷰어에게 "잘 봐줘"라고 하면 그럴듯한 합격을 준다. 대신 각 리뷰어에게 **반증을 기본값으로** 준다 —
"이 결함 주장이 틀렸음을 파일을 열어 직접 보여라, 못 보이면 결함 인정". 통과 기준을 높여 거짓 양성을 거른다.

## 루프 구조

```
라운드 시작
  │
  ▼  Find    4개 차원 비평가 동시 실행 (각자 자기 차원만 공격)
  │            harness-truth · mino-arch-fit
  │            a11y-coverage · test-quality
  │
  ▼  Dedup   이미 본 발견 제거 (제목+파일 키)
  │
  ▼  Verify  각 신규 발견을 독립 검증자 3명이 반증 시도
  │            3표 중 2표 이상 생존 → confirmed
  │
  ▼  채택    confirmed만 사람/메인 에이전트가 수정 반영
  │
  └─ 새 발견이 0건인 라운드가 2회 연속 → 종료 (loop-until-dry)
```

핵심 설계 두 가지:
- **차원 분리(multi-modal)**: 한 명이 다 보면 놓친다. 문서-코드 약속 일치(harness-truth), 레이어 적합성,
  접근성 커버리지, 테스트 품질을 각각 다른 눈이 본다.
- **반증 우선(adversarial verify)**: 검증자는 결함을 죽이는 게 기본 임무다. 살아남아야 진짜다.

## 실행

Claude Code에서 Workflow 툴로 돌린다.

```
Workflow({ scriptPath: "workflows/adversarial-harden.js" })
```

스크립트는 `{ confirmed: [...] }`를 반환한다. 각 항목은 `{ title, file, detail, fix, severity }`.
이 목록을 받아 메인 에이전트(또는 사람)가 수정에 반영하고, 오판 추적이 가능하도록 **차원별로 커밋을 분리**한다.

## 왜 수정까지 자동화하지 않나

검증자가 confirmed로 올린 결함도 **오판일 수 있다**. 수정을 자동 적용하면 오판이 산출물에 그대로 들어간다.
그래서 워크플로우는 "확정 결함 목록"까지만 내고, 반영은 사람이 diff로 판단할 수 있는 지점에 둔다.
(전역 규칙 "검증은 수렴할 때까지 반복, 수정 후 커밋"과 같은 취지.)

## 확장 아이디어

- 차원 추가: `DIMENSIONS` 배열에 항목을 더한다(예: 보안, 지역화, 성능).
- 검증 강도: 검증자 수를 3 → 5로, 생존 임계치를 2 → 3으로 올려 더 엄격하게.
- 토큰 예산 연동: `budget.total`이 잡혀 있으면 라운드 수를 예산에 맞춰 늘리는 루프로 바꿀 수 있다.
